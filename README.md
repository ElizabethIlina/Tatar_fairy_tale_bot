# Telegram-бот для генерации татарских волшебных сказок

Telegram-бот генерирует татарские волшебные сказки на основе пользовательских параметров: главного героя, злодея и двух помощников.  
Сказка создается с помощью OpenAI API по заранее подготовленному системному промпту, который задает жанровые, структурные и культурные ограничения татарской народной сказки.

## Возможности

Бот позволяет:

- запустить сценарий через команду `/start` или кнопку **«✨ Начать сказку»**;
- ввести главного героя сказки;
- выбрать злодея из предложенного списка или указать свой вариант;
- выбрать двух помощников из предложенного списка или указать собственные варианты;
- сгенерировать татарскую волшебную сказку;
- читать сказку по частям: сначала отображается первая половина, затем пользователь может нажать кнопку **«Читать далее...»**.

## Логика работы

После запуска бот просит пользователя указать четыре элемента сказки:

1. Главного героя.
2. Злодея.
3. Первого помощника.
4. Второго помощника.

После сбора всех данных бот передает их в системный промпт и отправляет запрос к OpenAI API.  
Сгенерированная сказка делится на две части, чтобы пользователю было удобнее читать длинный текст в Telegram.

## Описание файла `Tatar_fairy_tale_bot_code.py`

Файл `Tatar_fairy_tale_bot_code.py` является основным файлом проекта. В нем реализована вся логика Telegram-бота: запуск, обработка пользовательских сообщений, выбор персонажей, формирование промпта, обращение к OpenAI API и отправка сгенерированной сказки пользователю.

### Основные части файла

#### 1. Импорт библиотек

В начале файла подключаются необходимые библиотеки:

- `asyncio` — для асинхронного запуска бота;
- `os` — для работы с переменными окружения;
- `python-dotenv` — для загрузки токенов из файла `.env`;
- `aiogram` — для работы с Telegram Bot API;
- `openai` — для обращения к OpenAI API.

#### 2. Загрузка переменных окружения

Файл использует переменные окружения:

- `TELEGRAM_TOKEN` — токен Telegram-бота;
- `OPENAI_API_KEY` — API-ключ OpenAI.

Они загружаются из `.env`-файла. После этого создаются объекты:

- `bot` — экземпляр Telegram-бота;
- `dp` — диспетчер aiogram;
- `client` — асинхронный клиент OpenAI.

#### 3. Состояния диалога

Класс `SetupStory` описывает этапы пользовательского сценария. Бот последовательно ожидает:

1. ввод главного героя;
2. выбор злодея;
3. ввод собственного злодея, если пользователь выбрал свой вариант;
4. выбор первого помощника;
5. ввод собственного первого помощника;
6. выбор второго помощника;
7. ввод собственного второго помощника.

Такой подход позволяет боту вести пользователя по шагам и сохранять промежуточные ответы.

#### 4. Разбиение длинного текста

Функция `split_text()` нужна для отправки длинных сообщений в Telegram.  
Так как Telegram ограничивает длину одного сообщения, функция делит длинный текст на несколько частей, стараясь разрывать его по абзацам.

#### 5. Клавиатуры

В файле реализованы два типа клавиатур:

- клавиатура запуска с кнопкой **«✨ Начать сказку»**;
- inline-клавиатуры для выбора злодея и помощников.

Пользователь может выбрать готовый вариант из списка или нажать **«✏️ Свой вариант»** и ввести собственного персонажа.

#### 6. Приветственное сообщение

Переменная `WELCOME_TEXT` содержит приветственный текст, который объясняет пользователю, что нужно сделать для генерации сказки.  
Бот просит выбрать четыре элемента:

1. главного героя;
2. злодея;
3. первого помощника;
4. второго помощника.

#### 7. Системный промпт

Функция `build_system_prompt()` формирует системный промпт для генерации сказки.  
В него подставляются выбранные пользователем значения:

- `{hero}` — главный герой;
- `{villain}` — главный злодей;
- `{helper1}` — первый помощник;
- `{helper2}` — второй помощник.

Промпт задает правила генерации татарской волшебной сказки: объем, структуру из трех частей, языковые формулы, ограничения по мифологии, запреты на западное фэнтези, требования к конфликту, помощникам и финалу.

#### 8. Генерация и отправка сказки

Функция `generate_and_send()` отвечает за генерацию сказки. Она:

1. получает данные пользователя из состояния;
2. показывает пользователю выбранных персонажей;
3. формирует системный промпт;
4. отправляет запрос к OpenAI API;
5. получает сгенерированный текст;
6. делит сказку на две части;
7. отправляет первую часть;
8. добавляет кнопку **«Читать далее...»**.

Вторая часть сказки сохраняется во временном состоянии пользователя и отправляется после нажатия кнопки.

#### 9. Запуск сценария

Функция `start_story()` очищает предыдущее состояние пользователя, отправляет приветственное сообщение и переводит бота в состояние ожидания главного героя.

Сценарий можно запустить двумя способами:

- командой `/start`;
- кнопкой **«✨ Начать сказку»**.

#### 10. Обработчики шагов

В файле реализованы обработчики для всех этапов:

- получение главного героя;
- выбор злодея;
- ввод собственного злодея;
- выбор первого помощника;
- ввод собственного первого помощника;
- выбор второго помощника;
- ввод собственного второго помощника;
- отправка продолжения сказки.

Также добавлены защитные обработчики: если пользователь вместо нажатия кнопки вводит текст на этапе выбора, бот просит выбрать вариант из кнопок.

#### 11. Кнопка «Читать далее...»

После генерации бот сначала отправляет только первую часть сказки.  
Чтобы увидеть продолжение, пользователь нажимает кнопку **«Читать далее...»**. После этого бот отправляет вторую часть, очищает состояние и предлагает начать новую сказку.

#### 12. Fallback-обработчик

Если пользователь пишет боту вне активного сценария, срабатывает общий обработчик. Он предлагает начать сказку с помощью кнопки или команды `/start`.

#### 13. Запуск бота

В конце файла находится функция `main()`, которая запускает polling.  
При запуске файла напрямую выполняется:

```python
asyncio.run(main())
```
